## Library

### v0.1.0 — Alpha (shipped)

The initial alpha release establishes the workflow orchestration framework and
six built-in evaluation workflows. See [CHANGELOG.md](CHANGELOG.md) for the
full list.

- [x] Workflow orchestration framework — registry, task runner, pipeline configuration
- [x] Data cleaning workflow (outlier + duplicate detection)
- [x] Data analysis workflow (statistical summaries)
- [x] Dataset splitting workflow (stratified and random strategies)
- [x] Drift detection workflow (classwise drift)
- [x] Out-of-distribution (OOD) detection workflow
- [x] Prioritization workflow (dataset sample ranking)
- [x] Interactive TUI app + simple CLI config builder
- [x] Disk-backed and in-memory result caching
- [x] HuggingFace, MAITE, TorchVision, ImageFolder, COCO, YOLO dataset adapters
- [x] ONNX-based embedding extraction
- [x] Multi-variant Docker containers (CPU, CUDA 11.8 / 12.4 / 12.8) with Cosign signing
- [x] Sphinx documentation with tutorial notebooks
- [x] GitLab CI/CD with SAST / dependency / secret scanning + SBOM
- [x] 90% test coverage enforcement, 100% type completeness on public API

## Interoperability

### Currently supported

- **Datasets:** HuggingFace Vision, COCO, YOLO, TorchVision, ImageFolder, raw MAITE objects
- **Models / extractors:** ONNX, PyTorch, BoVW (SIFT), Flatten, Uncertainty
- **MAITE protocols:** consumes MAITE-compatible image classification and object detection datasets

### Planned / under consideration — TBD

- [ ] **VisDrone Static + Video dataset formats** (IR-3.2-S-3, IR-3.3-S-3) — decision pending; document as exception if descoped.
- [ ] **TFLite model format** (IR-3-S-2) — decision pending; document as exception if descoped.
- [ ] **Video workflows** (IR-3.3-S-*) — coordinated with `dataeval` v2.0 video roadmap.

---

## Out of scope (current line)

- Full motion video evaluation — deferred to align with `dataeval` v2.0 (Dec 2026).
- Web-service / REST deployment mode — `dataeval-flow` ships as a batch container, not a long-running service. IR-2.3-H-2, IR-2.3-S-1, IR-2.4-S-1 are not applicable.

## How to update this roadmap

This file should be revised at least every six months per DR-2.2-H-3. When
updating:

1. Bump **Last updated** at the top.
2. Move shipped items from "planned" to a release section with `[x]`.
3. Push out targets that slipped, with a one-line note explaining the slip.
4. Add new planned work under the appropriate version heading.
